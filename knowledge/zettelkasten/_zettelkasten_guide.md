# Zettelkasten知識ベース

このディレクトリは、Zettelkasten方式で知識を管理します。

## Zettelkastenとは

Zettelkastenは、ドイツの社会学者ニクラス・ルーマンが開発した知識管理システムです。
小さなノート（Zettel）を相互にリンクさせることで、知識のネットワークを構築します。

## ノートの命名規則

各ノートは、**説明（slug）を先頭**に、タイムスタンプIDを末尾に配置します：

```
description_YYYYMMDDHHMMSS.md
```

例：`kaggle_basics_20240101123045.md`

### 同じ日に複数ノートを作成する場合

原則として `YYYYMMDDHHMMSS` により衝突確率は極小です（複数人運用＋Gitでも安全性が高い）。

それでも「同一秒に作成」などで衝突した場合は、末尾にサフィックスを付けて回避します（衝突が発生すれば、マージの際にコンフリクトが生じる）：

```
description_YYYYMMDDHHMMSS.md      ← 基本形
description_YYYYMMDDHHMMSS_a.md    ← 同一秒で2つ目（衝突回避）
description_YYYYMMDDHHMMSS_b.md    ← 同一秒で3つ目
...
```

**理由**: Obsidianのグラフビューでノート名が見やすくなります。

## リンクの作成

Obsidianでは、`[[ファイル名]]` の形式でリンクを作成できます。
これにより、知識間の関係性を可視化できます。

### リンクの書き方

```markdown
# 基本形式（ファイル名をそのまま表示）
[[feature_engineering_20240102]]

# エイリアス形式（表示名を指定）
[[feature_engineering_20240102|特徴量エンジニアリング]]
```

エイリアス形式を使うと、リンクが読みやすくなります。
`|` の後に書いた文字が表示され、クリックすると指定したファイルに飛びます。

## タスクへの変換

知識ノートからタスクを生成する際は、以下の手順を踏みます：

1. 知識ノートを読み、実装可能なタスクを特定
2. `knowledge/inbox/` にタスク/アイデア/メモを追加（共通Inbox）
3. タスクとして採用する場合は `knowledge/tasks/{active|waiting|someday}/` に **コピーしてSSOT化**し、元のInboxファイルは `knowledge/inbox/archive/` に移動（入力ログとして保持）。必要に応じてメタデータに `related_notes` を記録
4. タスクをJSON形式に変換してマルチエージェントシステムに引き渡し

## 補足：既存ノートとの互換

過去に作られたノートには `id: 20240101` のような「日付のみ」のIDが混在しています。  
本プロジェクトでは **今後の新規作成から `YYYYMMDDHHMMSS` に統一**し、既存ノートは段階的に整備（必要なものから更新）します。

## フィードバックループ

1. **知識収集**: 新しい知識をノートとして記録
2. **タスク生成**: 知識から実装可能なタスクを生成
3. **実験実行**: マルチエージェントシステムで実験を実行
4. **学びの記録**: 実験結果を知識ノートに反映
5. **新しい知識**: 学びから新しい知識ノートを作成

この循環により、知識とタスクが有機的に成長していきます。
